# Search Context And Query Fan-Out

Use this reference when planning external search.

Use query fan-out as a research pattern for understanding what evidence a buyer question needs. Do not use it as a reason to create many thin exact-match pages.

## Pattern

```text
Buyer question:
Official-source queries:
Third-party-source queries:
Community-source queries:
Competitor queries:
Gaps to check on official site:
```

## Trust Question Example

```text
Buyer question:
- Is [brand] safe or reliable?

Official-source queries:
- [brand] safety
- [brand] regulation
- [brand] security
- [brand] legal documents

Third-party-source queries:
- [brand] review
- [brand] trustpilot
- [brand] complaints

Community-source queries:
- [brand] reddit
- [brand] withdrawal issue
- [brand] support experience

Competitor queries:
- [brand] vs [competitor]
- [category] safest providers

Gaps to check on official site:
- trust center
- risk disclosure
- complaint process
- eligibility restrictions
- support/help center coverage
```

## Pricing Question Example

```text
Buyer question:
- Which plan/account/product is best for my use case?

Official-source queries:
- [brand] pricing
- [brand] plans
- [brand] fees
- [brand] account comparison

Third-party-source queries:
- [brand] pricing review
- [category] pricing comparison

Competitor queries:
- [competitor] pricing
- [brand] alternatives pricing

Gaps to check on official site:
- comparison table
- total cost explanation
- example scenarios
- fee exclusions or limitations
```

## Rules

- Use fan-out to understand evidence needs, not to mass-produce pages.
- Prefer improving existing high-value pages over creating thin exact-match pages.
- Record which external sources AI or users are likely to trust.
- Check whether official profiles, review platforms, communities, directories, and regulators use the same brand/category language as the website.
